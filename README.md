## 2.小记
### 2.1碰撞检测
在进行碰撞检测时，可以在update子弹时进行检测也可以在update玩家和敌人时进行检测

### 2.2序列帧动画
    这里使用了一个结构体来存储帧动画的信息，包括：当前帧，总帧数，当前时间以及FPS
    在处理动画时，还是老样子先加载模板，这里通过总帧数和当前帧的顺序来进行切换动画（以下都在update函数中实现）
    1. 当前帧：当飞机死亡时，记录当前飞机死亡时间，当前帧序号 = （当前时间 - 死亡时间）* FPS / 1000 ； 
        其中，此FPS为帧动画的帧率 ， 因为时间的单位为毫秒，因此这里要除1000
    2. 总帧数：图像的长度 / 宽度
    在进行渲染时，通过当前帧的序号来渲染指定的帧
### 2.3物品掉落
    物品掉落的逻辑和其他的是一样的，在这里注意物品的渲染应当晚于爆炸的特效
    在进行道具的生成时，注意方向是随机给出的，并转换为对应的方向向量
### 2.4对象创建逻辑
    敌人的生成是通过std::list链表联系起来的(子弹等其他对象也是类似的)，其中，第一个函数创建敌人
    另外两个函数负责检查list
    1）如果list中有成员，就负责更新其状态
    2）如果list中有成员，就负责根据其状态来进行渲染
### 2.5播放音乐
    一共有两种处理方式
    Mix_Music ：播放时解码，适合处理背景音乐
    Mix_Chunk ：载入时解码，适合处理音效
    首先要进行初始化
    在初始化时，背景音乐单独处理，而音效可以通过 std::map 来进行存储，这里key值为音效名称，value为对应的音效
    在对应触发的位置添加音效
